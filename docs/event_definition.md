# Event Definition

## 1. 문서 목적

이 문서는 종합 이커머스 사용자 여정 분석을 위한 초기 8개 이벤트의 정의 초안이다.

이벤트 설계는 GA4 스타일 이벤트명을 기준으로 하며, 사용자 행동 로그에서 유입, 탐색, 상품 조회, 장바구니, 체크아웃, 구매, 구매 이후 리뷰 행동을 재구성할 수 있도록 정의한다.

## 2. 공통 이벤트 필드

모든 이벤트는 아래 공통 필드를 가진다고 가정한다.

| 필드 | 설명 | 필수 여부 |
|---|---|---|
| `event_id` | 이벤트 고유 식별자 | 필수 |
| `event_name` | 이벤트 이름 | 필수 |
| `event_timestamp` | 이벤트 발생 시각 | 필수 |
| `event_date` | 이벤트 발생 날짜 | 필수 |
| `user_id` | 사용자 식별자 | 필수 |
| `session_id` | 세션 식별자 | 필수 |

이벤트별 파라미터는 공통 필드 외에 이벤트의 의미를 설명하기 위해 추가로 수집하는 값이다.

이벤트별 필수 파라미터는 행동 발생 사실과 외부 엔티티를 연결하기 위한 최소 식별자를 우선 저장한다. 상품명, 목록명, 주문 상품 상세 같은 설명성 정보는 향후 별도 참조 테이블 또는 상세 테이블에서 관리할 수 있으므로 필수 파라미터에서 제외한다.

`event_date`는 `event_timestamp`에서 파생 가능한 값이지만, 일자별 집계와 날짜 기준 필터링을 쉽게 하기 위해 공통 필드에 포함한다.

`device_type`, `traffic_source`, `platform`은 공통 이벤트 필드로 확정하지 않는다. 해당 값은 세션 속성 또는 이벤트별 선택 파라미터 후보로 남긴다.

현재 프로젝트는 국내 이커머스 서비스를 가정하므로 금액 단위는 KRW로 고정한다. 따라서 이벤트별 파라미터에서 `currency`는 별도로 수집하지 않으며, `value`는 원화 기준 금액으로 정의한다.

## 3. 이벤트 정의

### 3.1 session_start

| 항목 | 내용 |
|---|---|
| event_name | `session_start` |
| 이벤트 설명 | 사용자의 방문 세션이 시작되었음을 나타내는 이벤트 |
| 발생 조건 | 사용자가 서비스에 진입하여 새로운 세션이 생성될 때 발생 |
| 분석 목적 | 세션 수 산출, 유입 경로 분석, 세션 기준 사용자 여정 재구성 |
| 필수 파라미터 | `event_timestamp`, `user_id`, `session_id` |
| 선택 파라미터 | `traffic_source`, `medium`, `campaign`, `device_type`, `platform`, `landing_page` |
| 주의할 정합성 검증 포인트 | 동일 `session_id` 내에서 `session_start`가 중복 발생하지 않는지 확인한다. 동일 세션의 다른 이벤트보다 늦게 기록되지 않았는지 확인한다. `user_id`와 `session_id`가 누락되지 않아야 한다. |

### 3.2 search

| 항목 | 내용 |
|---|---|
| event_name | `search` |
| 이벤트 설명 | 사용자가 검색어를 입력하고 검색을 실행한 이벤트 |
| 발생 조건 | 검색창, 검색 결과 페이지, 자동완성 선택 등에서 검색 요청이 완료될 때 발생 |
| 분석 목적 | 검색 중심 탐색 비중 분석, 검색어별 상품 관심도 분석, 검색 이후 상세 조회 및 구매 전환 분석 |
| 필수 파라미터 | `event_timestamp`, `user_id`, `session_id`, `search_term` |
| 선택 파라미터 | `search_result_count`, `search_type`, `category_id`, `sort_option`, `filter_options` |
| 주의할 정합성 검증 포인트 | `search_term`이 비어 있거나 공백만 있는지 확인한다. 검색 결과 수가 음수로 기록되지 않아야 한다. 동일 사용자가 짧은 시간 내 같은 검색어를 반복 실행한 경우 중복 이벤트 여부를 확인한다. |

### 3.3 view_item_list

| 항목 | 내용 |
|---|---|
| event_name | `view_item_list` |
| 이벤트 설명 | 사용자가 상품 목록 또는 검색 결과 목록을 조회한 이벤트 |
| 발생 조건 | 카테고리 목록, 검색 결과, 추천 상품 목록, 기획전 목록 등 상품 리스트가 화면에 노출될 때 발생 |
| 분석 목적 | 탐색 단계 분석, 목록 조회 후 상품 상세 조회 전환율 분석, 카테고리 또는 리스트별 관심도 분석 |
| 필수 파라미터 | `item_list_id` |
| 선택 파라미터 | `item_list_name` |
| 주의할 정합성 검증 포인트 | `item_list_id`가 누락되지 않아야 한다. `item_list_name`이 있는 경우 동일 `item_list_id`에 대해 일관된 목록명으로 기록되는지 확인한다. |

### 3.4 view_item

| 항목 | 내용 |
|---|---|
| event_name | `view_item` |
| 이벤트 설명 | 사용자가 상품 상세 페이지를 조회한 이벤트 |
| 발생 조건 | 상품 상세 페이지 또는 상품 상세 모달이 사용자에게 표시될 때 발생 |
| 분석 목적 | 상품 관심도 분석, 목록 조회 후 상세 조회 전환 분석, 상세 조회 이후 장바구니 및 구매 전환 분석 |
| 필수 파라미터 | `item_id` |
| 선택 파라미터 | `item_name` |
| 주의할 정합성 검증 포인트 | `item_id`가 누락되지 않아야 한다. `item_name`이 있는 경우 동일 `item_id`에 대해 일관된 상품명으로 기록되는지 확인한다. |

### 3.5 add_to_cart

| 항목 | 내용 |
|---|---|
| event_name | `add_to_cart` |
| 이벤트 설명 | 사용자가 상품을 장바구니에 담은 이벤트 |
| 발생 조건 | 상품 상세, 상품 목록, 추천 영역 등에서 장바구니 담기 액션이 성공적으로 완료될 때 발생 |
| 분석 목적 | 구매 의도 분석, 상품 상세 조회 후 장바구니 전환율 분석, 장바구니 진입 상품 및 카테고리 분석 |
| 필수 파라미터 | `item_id`, `quantity` |
| 선택 파라미터 | `item_name` |
| 주의할 정합성 검증 포인트 | `quantity`는 1 이상이어야 한다. `item_id`가 누락되지 않아야 한다. 같은 세션에서 `view_item` 없이 `add_to_cart`가 발생한 경우 허용 가능한 유입 경로인지 확인한다. |

### 3.6 begin_checkout

| 항목 | 내용 |
|---|---|
| event_name | `begin_checkout` |
| 이벤트 설명 | 사용자가 장바구니 또는 바로구매 흐름에서 주문 절차를 시작한 이벤트 |
| 발생 조건 | 주문서 화면 진입, 결제 단계 진입 등 체크아웃 프로세스가 시작될 때 발생 |
| 분석 목적 | 장바구니 이후 체크아웃 전환율 분석, 구매 퍼널 진입 사용자 분석, 체크아웃 단계 이탈 분석 |
| 필수 파라미터 | `checkout_id`, `value` |
| 선택 파라미터 | `items` |
| 주의할 정합성 검증 포인트 | `checkout_id`가 누락되지 않아야 한다. `value`는 원화 기준 금액이며 0 이상이어야 한다. `items`가 있는 경우 상품 ID가 비어 있지 않은지 확인한다. `items`는 향후 `checkout_items` 또는 `cart_items` 테이블로 분리할 수 있다. 일반 장바구니 구매 흐름에서는 앞선 `add_to_cart` 이벤트와 연결 가능한지 확인한다. |

### 3.7 purchase

| 항목 | 내용 |
|---|---|
| event_name | `purchase` |
| 이벤트 설명 | 사용자의 주문과 결제가 완료된 이벤트 |
| 발생 조건 | 결제 승인 또는 주문 완료 처리가 성공적으로 끝났을 때 발생 |
| 분석 목적 | 최종 구매 전환율 분석, 매출 분석, 체크아웃 이후 구매 완료 전환 분석, 구매 상품 및 카테고리 분석 |
| 필수 파라미터 | `order_id`, `value` |
| 선택 파라미터 | `items` |
| 주의할 정합성 검증 포인트 | `order_id`가 누락되지 않아야 하며 중복 구매로 기록되지 않아야 한다. `value`는 원화 기준 금액이며 상품 금액, 배송비, 세금, 할인 정책과 일관되어야 한다. `items`가 있는 경우 상품 ID가 비어 있지 않은지 확인한다. `items`는 향후 `order_items` 테이블로 분리할 수 있다. 같은 세션 또는 합리적인 시간 범위 내의 `begin_checkout`과 연결 가능한지 확인한다. |

### 3.8 review_write

| 항목 | 내용 |
|---|---|
| event_name | `review_write` |
| 이벤트 설명 | 사용자가 구매한 상품에 대해 리뷰를 작성한 이벤트 |
| 발생 조건 | 리뷰 작성이 저장 또는 제출 완료되었을 때 발생 |
| 분석 목적 | 구매 이후 행동 분석, 리뷰 작성률 분석, 상품별 후기 참여도 분석 |
| 필수 파라미터 | `event_timestamp`, `user_id`, `session_id`, `item_id`, `order_id`, `review_id`, `rating` |
| 선택 파라미터 | `review_length`, `has_photo`, `has_video`, `category_id`, `category_name` |
| 주의할 정합성 검증 포인트 | 리뷰는 구매 이후 행동이므로 `order_id`가 실제 `purchase` 이벤트와 연결 가능한지 확인한다. `rating`은 정의된 평점 범위 안에 있어야 한다. 동일 `review_id`가 중복 기록되지 않아야 한다. |

## 4. 공통 정합성 검증 기준

- 모든 이벤트는 허용된 `event_name` 목록에 포함되어야 한다.
- 모든 이벤트는 `event_timestamp`, `user_id`, `session_id`를 가져야 한다.
- 동일 `session_id` 내 이벤트 시간 순서가 역전되지 않아야 한다.
- 퍼널 분석 대상 이벤트는 일반적으로 `session_start` 이후에 발생해야 한다.
- 구매 퍼널은 `view_item` -> `add_to_cart` -> `begin_checkout` -> `purchase` 흐름을 기준으로 검증하되, 바로구매 등 예외 흐름은 별도 정책으로 정의해야 한다.
- `purchase`는 `order_id` 기준 중복 여부를 반드시 확인해야 한다.
- `review_write`는 핵심 구매 퍼널에는 포함하지 않고 구매 이후 행동 분석 이벤트로 분리한다.
