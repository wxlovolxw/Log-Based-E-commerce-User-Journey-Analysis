
# Project Plan

## 1. 프로젝트 개요

### 1.1 배경

이 프로젝트는 종합 이커머스 도메인에서 발생하는 사용자 행동 로그를 기반으로 유저 여정을 분석하기 위한 분석 프로젝트이다.

사용자의 유입, 탐색, 상품 조회, 장바구니, 구매 전환, 이탈 흐름을 이벤트 로그 단위로 추적하고, 전환 병목과 행동 패턴을 파악하는 것을 목표로 한다.

### 1.2 분석 범위

- 도메인: 종합 이커머스
- 분석 방식: 로그 기반 사용자 행동 분석
- 주요 분석 대상: 사용자 이벤트 로그
- 주요 여정: 유입 -> 탐색 -> 상품 조회 -> 장바구니 -> 구매
- 주요 관심사: 전환율, 이탈 지점, 사용자 행동 흐름, 이벤트 정합성

### 1.3 현재 상태

- 저장소 기본 구조가 생성되어 있다.
- `data/`, `docs/`, `notebooks/`, `sql/` 디렉터리가 존재한다.
- 프로젝트 계획 문서, 의사결정 기록 문서, 이벤트 정의서를 작성했다.
- 1차 엔티티 정의서를 작성하고 주요 엔티티의 역할과 속성 초안을 정리했다.
- 엔티티 간 관계 분석을 1차 완료했다.
- 1차 정합성 검증 항목 초안을 작성하고 별도 문서로 분리했다.
- 실제 데이터, SQL 쿼리, 분석 노트북은 아직 작성되지 않았다.

## 2. 프로젝트 목표

### 2.1 핵심 목표

- 종합 이커머스 사용자 행동 로그를 분석 가능한 형태로 정의한다.
- GA4 스타일 이벤트 설계를 기준으로 이벤트 체계를 정리한다.
- 사용자 여정의 주요 단계별 전환과 이탈을 분석한다.
- 데이터 정합성을 검증하여 분석 결과의 신뢰도를 확보한다.
- 최종적으로 유저 여정의 병목과 개선 가능 지점을 도출한다.

### 2.2 기대 산출물

- 프로젝트 계획 문서
- 설계 의사결정 기록 문서
- 이벤트 정의서
- 데이터 모델 정의
- 정합성 검증 규칙 문서
- 정합성 검증 쿼리
- 사용자 여정 분석 SQL
- EDA 및 분석 노트북
- 최종 분석 결과 요약

### 2.3 성공 기준

- 주요 사용자 행동 이벤트가 일관된 기준으로 정의되어 있다.
- 이벤트 로그에서 사용자 여정 단계를 재구성할 수 있다.
- 핵심 퍼널의 전환율과 이탈률을 산출할 수 있다.
- 이벤트 누락, 중복, 순서 오류 등 정합성 이슈를 검증할 수 있다.
- 분석 결과가 비즈니스 질문에 직접 답할 수 있다.

## 3. 도메인 정의

### 3.1 이커머스 유저 여정

종합 이커머스의 기본 유저 여정은 사용자가 서비스에 유입된 뒤 상품을 탐색하고, 상품 상세를 확인하며, 장바구니 또는 구매 단계로 이동하는 흐름으로 정의한다.

초기 기준 여정은 다음과 같다.

1. 서비스 유입
2. 상품 목록 또는 검색 결과 탐색
3. 상품 상세 조회
4. 장바구니 담기
5. 주문 시작
6. 결제 완료

### 3.2 주요 엔티티

| 엔티티 | 설명 | 현재 상태 |
|---|---|---|
| `users` | 서비스를 이용하는 사용자 | 역할 정의 완료, 속성 초안 완료 |
| `sessions` | 사용자의 방문 단위 | 역할 정의 완료, 속성 초안 완료 |
| `categories` | 상품 분류 체계 | 역할 정의 완료, 속성 초안 완료 |
| `products` | 판매 상품 | 역할 정의 완료, 속성 초안 완료, 이벤트 `item_id`와 `product_id` 매핑 정리 |
| `orders` | 주문 또는 결제 결과 | 역할 정의 완료, 속성 초안 완료 |
| `order_items` | 주문에 포함된 개별 상품 | 역할 정의 완료, 속성 초안 완료 |
| `reviews` | 구매 이후 작성된 상품 리뷰 | 역할 정의 완료, 속성 초안 완료 |
| `event_logs` | 사용자의 행동 로그 단위 | 이벤트 정의 완료, 공통 필드 정의 완료 |

### 3.3 주요 이벤트

초기 이벤트 설계는 GA4 스타일 이벤트 설계를 따른다. 이벤트는 `event_name`, `event_timestamp`, `user_id`, `session_id`, 이벤트 파라미터를 중심으로 정의한다.

| 이벤트 | 설명 | 비고 |
|---|---|---|
| `session_start` | 세션 시작 | 유입 분석 기준 |
| `search` | 검색 실행 | 검색 행동 분석 |
| `view_item_list` | 상품 목록 조회 | 탐색 단계 |
| `view_item` | 상품 상세 조회 | 상품 관심도 기준 |
| `add_to_cart` | 장바구니 담기 | 구매 의도 기준 |
| `begin_checkout` | 주문 시작 | 구매 퍼널 진입 |
| `purchase` | 구매 완료 | 최종 전환 |
| `review_write` | 리뷰 작성 | 구매 이후 행동 분석 |

### 3.4 주요 지표

- 세션 수
- 사용자 수
- 상품 조회 수
- 장바구니 전환율
- 체크아웃 전환율
- 구매 전환율
- 단계별 이탈률
- 이벤트 누락률
- 이벤트 중복률

## 4. 핵심 분석 질문

### 4.1 유입 및 탐색

- 사용자는 어떤 경로로 서비스에 유입되는가?
- 세션 시작 후 첫 행동은 무엇인가?
- 탐색 행동은 검색 중심인가, 카테고리/목록 조회 중심인가?

### 4.2 상품 조회 및 장바구니

- 상품 목록 조회 후 상세 조회로 얼마나 전환되는가?
- 상세 조회 후 장바구니 담기로 얼마나 전환되는가?
- 특정 상품 또는 카테고리에서 이탈이 집중되는가?

### 4.3 구매 전환

- 장바구니 담기 이후 체크아웃으로 얼마나 이동하는가?
- 체크아웃 이후 구매 완료까지의 전환율은 어느 정도인가?
- 구매 전환에 영향을 줄 수 있는 주요 행동 패턴은 무엇인가?

### 4.4 이탈 및 병목

- 유저 여정에서 이탈이 가장 크게 발생하는 단계는 어디인가?
- 이벤트 순서상 비정상 흐름은 존재하는가?
- 정합성 이슈가 분석 결과에 영향을 줄 수 있는가?

## 5. 현재 설계

### 5.1 이벤트 설계

GA4 스타일 이벤트 설계를 채택한다.

기본 방향은 다음과 같다.

- 사용자 행동을 이벤트 단위로 기록한다.
- 이벤트 이름은 행동을 설명하는 명확한 동사형 또는 GA4 권장 이벤트명을 사용한다.
- 이벤트 공통 필드는 `event_name`, `event_timestamp`, `user_id`, `session_id`를 중심으로 구성한다.
- 상품, 주문, 검색어, 유입 경로 등은 이벤트 파라미터로 관리한다.
- 퍼널 분석이 가능하도록 주요 단계 이벤트를 명시적으로 정의한다.

### 5.2 데이터 모델

초기 데이터 모델은 이벤트 로그 중심으로 시작하되, 이벤트 분석에 필요한 1차 참조 엔티티를 함께 도출한다.

| 테이블 | 목적 | 현재 상태 |
|---|---|---|
| `event_logs` | 사용자 행동 이벤트 원천 로그 | 이벤트 정의 완료, 공통 필드 정의 완료, 이벤트별 파라미터 정의 완료 |
| `users` | 사용자 속성 정보 | 엔티티 도출 완료, 역할 정의 완료, 속성 초안 완료 |
| `sessions` | 사용자 방문 단위 정보 | 엔티티 도출 완료, 역할 정의 완료, 속성 초안 완료 |
| `categories` | 상품 카테고리 정보 | 엔티티 도출 완료, 역할 정의 완료, 속성 초안 완료 |
| `products` | 상품 속성 정보 | 엔티티 도출 완료, 역할 정의 완료, 속성 초안 완료 |
| `orders` | 주문 결과 정보 | 엔티티 도출 완료, 역할 정의 완료, 속성 초안 완료 |
| `order_items` | 주문 상품 상세 정보 | 엔티티 도출 완료, 역할 정의 완료, 속성 초안 완료 |
| `reviews` | 상품 리뷰 정보 | 엔티티 도출 완료, 역할 정의 완료, 속성 초안 완료 |

현재 단계에서는 물리적 테이블 설계가 아니라 논리적 엔티티와 주요 속성 초안을 정리한다. 엔티티 간 관계 분석은 1차 완료했으며, 정합성 검증 항목은 1차 초안을 작성했다.

### 5.3 정합성 검증 계획

정합성 검증은 분석 이전 단계에서 수행한다.

1차 검증 규칙은 `data_quality_rules.md`에 문서화한다.

초기 검증 항목은 다음과 같다.

- 필수 필드 누락 여부
- 이벤트명 허용 목록 위반 여부
- 사용자 ID 또는 세션 ID 누락 여부
- `event_logs.user_id`와 `sessions.user_id` 일치 여부
- 이벤트 타임스탬프 역전 여부
- 동일 이벤트 중복 발생 여부
- 구매 이벤트의 주문 ID 누락 여부
- 퍼널 순서 위반 여부
