# A/B 테스트 설계안

## 1. 문서 개요

본 문서는 SQL 퍼널 분석, 구매 전환 행동 분석, Logistic Regression baseline 결과를 바탕으로 작성한 A/B 테스트 설계안이다.

현재 데이터는 synthetic data이므로 실제 서비스 효과를 입증한 것이 아니다. 따라서 아래 내용은 실제 실험 결과가 아니라, 향후 실제 서비스에서 검증 가능한 분석 결과 기반 가설과 실험 설계안으로 해석한다.

## 2. 분석 근거 요약

### 2.1 퍼널 분석

| 단계 | 도달 세션 수 |
|---|---:|
| `session_start` | 1000 |
| `view_item` | 867 |
| `add_to_cart` | 578 |
| `begin_checkout` | 366 |
| `purchase` | 282 |

주요 전환율은 다음과 같다.

| 구간 | 전환율 |
|---|---:|
| `add_to_cart` -> `begin_checkout` | 63.32% |
| `begin_checkout` -> `purchase` | 77.05% |

### 2.2 행동 도달 여부별 구매 전환율

| 행동 도달 기준 | 구매 전환율 |
|---|---:|
| `add_to_cart` 도달 세션 | 48.79% |
| `begin_checkout` 도달 세션 | 77.05% |

### 2.3 모델링 결과

Logistic Regression baseline에서 구매 직전 행동인 `begin_checkout_count`, `has_begin_checkout`을 제외한 `without_checkout` 모델도 F1 0.8889, ROC-AUC 0.9834로 높은 구분 성능을 보였다.

`without_checkout` 모델 기준으로는 `event_count`, `has_add_to_cart`가 구매 전환 여부를 구분하는 주요 양의 신호로 나타났다. 이는 개별 coefficient를 단독으로 해석하기보다, 세션 내 전체 행동량과 장바구니 도달 여부가 구매 전환과 관련된 핵심 신호라는 방향으로 해석한다.

### 2.4 실험 후보 해석

퍼널과 모델링 결과를 함께 보면, 장바구니 도달 이후 결제 시작으로 넘어가는 구간이 실험 설계의 우선 후보이다. `add_to_cart`는 구매 전환과 관련된 주요 행동 신호이며, `add_to_cart`에서 `begin_checkout`으로 이동하는 구간에는 추가 개선 가설을 설정할 여지가 있다.

## 3. A/B Test 1: 장바구니 -> 결제 시작 전환 개선

### 3.1 문제 정의

`add_to_cart`에 도달한 세션 중 일부만 `begin_checkout`으로 이동한다. 장바구니에 상품을 담은 사용자가 결제 시작 단계로 넘어가도록 돕는 UI 및 정보 제공 개선안을 실험 후보로 설정한다.

### 3.2 가설

장바구니 페이지에서 결제 시작 CTA를 더 명확하게 노출하면 `begin_checkout` 전환율이 상승할 것이다.

### 3.3 Control

기존 장바구니 페이지를 유지한다.

### 3.4 Treatment

- 결제 시작 버튼을 더 명확하게 강조한다.
- 배송 및 결제 혜택 안내를 장바구니 페이지에 추가한다.
- 구매 불안 요소를 완화하는 문구를 추가한다.

### 3.5 Primary Metric

- `add_to_cart` 도달 세션 중 `begin_checkout` 전환율

### 3.6 Secondary Metrics

- `purchase` 전환율
- checkout 시작 후 `purchase` 전환율
- 평균 주문 금액 또는 `total_value`

### 3.7 Guardrail Metrics

- 이탈률
- 취소/환불률이 있다면 추후 포함
- 과도한 CTA로 인한 UX 악화 가능성

### 3.8 대상 세션

- `add_to_cart`에 도달한 세션

### 3.9 기대 해석

`begin_checkout` 전환율이 상승하고 `purchase` 전환율도 유지 또는 상승하면 개선안 채택 후보로 볼 수 있다. 반대로 `begin_checkout`은 증가했지만 `purchase` 전환율이나 guardrail metric이 악화된다면, CTA 강화가 실제 구매 의도 개선으로 이어졌는지 추가 검토가 필요하다.

## 4. A/B Test 2: 상품 탐색 -> 장바구니 추가 유도

### 4.1 가설

상품 상세 페이지에서 장바구니 CTA와 상품 정보 구조를 개선하면 `add_to_cart` 전환율이 상승할 것이다.

### 4.2 Control

기존 상품 상세 페이지를 유지한다.

### 4.3 Treatment

- 장바구니 CTA의 위치와 시각적 우선순위를 개선한다.
- 가격, 배송, 재고, 혜택 등 구매 판단에 필요한 정보를 더 명확하게 정리한다.
- 상품 상세 페이지에서 장바구니 추가까지의 행동 흐름을 단순화한다.

### 4.4 Primary Metric

- `view_item` 도달 세션 중 `add_to_cart` 전환율

### 4.5 Secondary Metrics

- `purchase` 전환율
- 상품 조회 수
- `unique_product_count`

### 4.6 Guardrail Metrics

- 짧은 체류 후 이탈 증가 여부
- 장바구니 추가는 증가했지만 구매 전환은 유지되는지 확인

### 4.7 기대 해석

`add_to_cart` 전환율이 상승하면서 `purchase` 전환율이 유지 또는 상승하면 탐색 단계 개선안으로 검토할 수 있다. 다만 장바구니 추가만 증가하고 구매 전환이 유지되지 않는다면, 상품 관심 표현은 늘었지만 구매 의도 개선으로 이어지지 않았을 가능성을 함께 검토해야 한다.

## 5. A/B Test 3: 구매 후 리뷰 작성 유도

### 5.1 분석 근거

synthetic data 기준 전체 주문 282건 중 리뷰 작성 주문은 112건이며, 주문 기준 리뷰 작성률은 39.72%였다.

### 5.2 가설

구매 완료 후 리뷰 작성 유도 메시지 또는 후속 알림을 제공하면 `review_write` 비율이 상승할 것이다.

### 5.3 Control

기존 구매 완료 및 구매 이후 커뮤니케이션 흐름을 유지한다.

### 5.4 Treatment

- 구매 완료 페이지에서 리뷰 작성 유도 메시지를 제공한다.
- 구매 이후 일정 시점에 리뷰 작성 후속 알림을 제공한다.
- 리뷰 작성의 기대 행동과 혜택을 명확하게 안내한다.

### 5.5 Primary Metric

- 주문 대비 리뷰 작성률

### 5.6 Secondary Metrics

- `same_purchase_session` 리뷰 비율
- `separate_post_purchase_session` 리뷰 비율
- 평균 리뷰 작성 지연 시간

### 5.7 Guardrail Metrics

- 과도한 알림으로 인한 사용자 불편 가능성

### 5.8 기대 해석

주문 대비 리뷰 작성률이 상승하고 사용자 불편 관련 guardrail metric이 악화되지 않는다면, 리뷰 작성 유도 정책의 채택 후보로 볼 수 있다. 다만 리뷰 작성은 구매 전환 이후 행동이므로, 구매 전환 개선 실험과는 목적을 분리해 해석한다.

## 6. 우선순위

| 우선순위 | 실험안 | 이유 |
|---:|---|---|
| 1 | 장바구니 -> 결제 시작 전환 개선 | 퍼널상 주요 이탈 구간이며, 모델링에서 `has_add_to_cart`가 주요 신호로 나타났고 구매 전환과 직접적으로 연결되는 구간이다. |
| 2 | 상품 탐색 -> 장바구니 추가 유도 | 구매 전환 이전의 핵심 행동인 `add_to_cart`를 늘리는 실험으로, 상위 퍼널 개선 가능성을 검토할 수 있다. |
| 3 | 구매 후 리뷰 작성 유도 | 구매 이후 행동 개선 실험으로 의미가 있으나, 구매 전환 개선과는 목적이 다르므로 후순위로 둔다. |

1순위는 장바구니 -> 결제 시작 전환 개선이다. 이 구간은 퍼널상 주요 이탈 구간이며, 모델링에서 `has_add_to_cart`가 주요 신호로 나타났고, 구매 전환과 직접적으로 연결되는 구간이다.

## 7. 한계

- synthetic data이므로 실제 사용자의 인과 효과를 확인한 것이 아니다.
- 모델링 결과는 실험 가설 생성을 위한 참고 자료이며, 실험 효과를 증명하지 않는다.
- 실제 A/B 테스트에서는 표본 크기, 실험 기간, 랜덤 배정 단위, 중복 노출 관리, 통계적 유의성 검정이 추가로 필요하다.
- 실제 서비스 적용 전에는 실험 대상, 노출 위치, 사용자 세그먼트, 측정 로그 정의를 별도로 확정해야 한다.
